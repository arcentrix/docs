## 数据流设计

### 任务执行数据流

```mermaid
sequenceDiagram
    participant U as 用户
    participant UI as Web UI
    participant LB as 负载均衡器
    participant S as Arcade Server
    participant DB as 数据库
    participant MQ as 消息队列
    participant A as Agent
    participant ST as 存储
    participant EXT as 外部系统
    
    Note over U,EXT: 流水线创建流程
    
    U->>UI: 创建流水线
    UI->>LB: POST /api/v1/pipelines
    LB->>S: 转发请求
    S->>DB: 保存流水线配置
    DB-->>S: 流水线 ID
    S->>MQ: 发布创建事件
    S-->>LB: 成功响应
    LB-->>UI: 流水线已创建
    UI-->>U: 显示成功
    
    Note over U,EXT: 流水线执行流程
    
    U->>UI: 触发流水线
    UI->>LB: POST /api/v1/pipelines/{id}/trigger
    LB->>S: 转发请求
    S->>DB: 创建流水线运行记录
    S->>MQ: 队列任务
    S->>EXT: 拉取代码
    S-->>LB: 运行 ID
    LB-->>UI: 执行已开始
    
    Note over U,EXT: 任务执行流程
    
    A->>S: FetchTask (gRPC)
    S->>MQ: 获取下一个任务
    MQ-->>S: 任务详情
    S->>DB: 更新任务状态
    S-->>A: 任务分配
    
    A->>A: 执行任务
    A->>S: ReportTaskStatus (RUNNING)
    A->>S: ReportTaskLog (流式)
    A->>ST: 上传构建产物
    
    A->>S: ReportTaskStatus (SUCCESS/FAILED)
    S->>DB: 更新任务状态
    S->>MQ: 发布任务完成事件
    S->>UI: WebSocket 状态更新
    UI-->>U: 实时状态更新
    
    Note over U,EXT: 通知流程
    
    S->>EXT: 发送通知
    EXT-->>U: Slack/邮件提醒
```

## 系统特性总结

### 核心特性

1. **高可用性**: 多节点集群，自动故障转移
2. **高性能**: 异步处理，连接池，缓存优化
3. **高扩展性**: 水平扩展，插件化架构
4. **高安全性**: 多层安全防护，数据加密
5. **易监控**: 全链路监控，实时告警
6. **易维护**: 容器化部署，自动化运维
